- Use `fd` instead of `find`
- Use `rg` (ripgrep) instead of `grep`
- Use `sg` (ast-grep) for structural code search
- Use `jq` for JSON filtering and transformation
- Use `yq` for YAML filtering

When writing Git commits, never use conventions. Do not prefix the message
with "feat:", "chore:", etc. Instead, write a single, elegant message
describing the change using up to 80 characters.
